How it actually works

01
One dropdown
Pick an AI worker or a human. That's the whole task assignment. No mode toggles. No configuration screens. "Who does this?" Done.
02
Streams, not boards
Tasks run top-to-bottom. One finishes, next starts automatically. No dragging cards across five columns so a PM can feel informed.
03
Workers ask when stuck
Job pauses. You answer inline. It picks up exactly where it left off. Unlike your last team lead, it doesn't pretend to understand and break everything.
04
Git happens automatically
Each stream gets its own branch. Each task auto-commits. Auto-revert on failure. Click "Create PR" when the stream is done. You make zero git decisions.
05
Watch them think
Live streaming as Claude reads files, writes code, runs tests. Push notifications when it's done or needs you. Not 2004-style dashboard refreshing.

Two ways to run it

On your machine
Run locally with Docker. Your code never leaves your machine. Good for solo developers who want an AI partner while they work.
On a VPS
AI workers grind tasks 24/7 while you sleep. Wake up to PRs. Good for teams who like mornings more than late nights.

Three steps. No onboarding call.

1
Clone it
Node, Docker, Claude subscription. That's the whole dependency list.
git clone git@github.com:ilyador/workstream.git && cd workstream && pnpm install
2
Start it
One command spins up Supabase, the API, and the frontend.
npx supabase start && npx supabase db reset && pnpm dev
3
Ship something
Point it at your repo. Create a task. Assign a worker. Go make coffee. Come back to a PR.
open http://localhost:3000
