Things we proudly don't have

Story Points
Estimation theater
You've never estimated correctly. Neither has anyone in the history of software. We just track what actually shipped.
Sprints
Arbitrary time boxes
Nothing magical happens every two weeks except a meeting where everyone lies about what they'll finish next.
Priority Levels
P0 P1 P2 P3 P4
Three options: critical, upcoming, or backlog. Drag to reorder within each. No spreadsheet of severity matrices that nobody reads.
Velocity Charts
Management pacifiers
A chart that goes up when you close tickets faster. Congratulations, you've incentivized splitting one task into twelve.

What it does instead

01
AI writes your code
Click Run. Claude reads your codebase, plans the approach, implements it, runs tests, and asks you to review. You approve or reject. No standups required.
plan → implement → verify → review
02
Workstreams, not boards
Sequential columns of tasks that execute top-to-bottom. When one finishes, the next starts automatically. Like a CI pipeline, but for your entire backlog.
03
It asks when it's stuck
Claude doesn't know your AWS credentials? Job pauses. You answer inline. It picks up where it left off. No babysitting, no crashed builds.
04
Git without the footgun
Each workstream gets its own git worktree — isolated from main, isolated from each other. Each task auto-commits on completion. When the workstream is done, click "Create PR." No git choices to make. It's all automatic. AI never improvises with your repo.
05
Auto-revert on failure
Every task gets a git checkpoint inside its worktree before execution. If something breaks, the worktree reverts to exactly where it was. Your main checkout is never touched. Failed tasks clean up after themselves.
06
Skills & image context
Type /skillname in a task description to inject Claude Code skills into the AI prompt, with autocomplete. Drag-drop screenshots, designs, or error captures — they're passed directly to Claude as visual context.
07
Team roles that make sense
Three roles: admin, dev, manager. Managers can create tasks and review work but can't trigger AI execution. Invite people by email — accounts are created automatically if they don't have one yet.
08
Realtime everything
Live job logs stream to your browser as Claude works. Task and job status sync instantly across team members via SSE and Supabase Realtime. Web push notifications when tasks complete, fail, or need input.
09
Comments & @mentions
Per-task comment threads with @mention notifications. Discuss work inline without leaving the board. Comments are for humans — AI reads the task description, not the thread.
10
Human tasks exist too
Not everything needs AI. Design reviews, meetings, manual QA — assign to a team member. It's a toggle, not a separate product.
11
Telegram bot
Create tasks, check project status, and get summaries from Telegram. Link a chat to a project and talk to it in plain English. Claude interprets and acts.
12
MCP server & CLI
Ask Claude Code "what should I work on?" from your terminal. Create tasks, reply to paused jobs, approve reviews — without opening a browser.
9 MCP tools
13
Runs anywhere you want
Run it locally and sync to a cloud database. Or throw it on a VPS with local Supabase and let AI grind tasks in isolated worktrees while you sleep. Daily DB backups with a single script.

Two ways to run it

On your machine
Run locally, sync through an online Supabase. Good for solo developers who want AI as a coding partner while they work.
On a VPS
Run on a server with local Supabase. AI grinds tasks 24/7 while you sleep. Good for teams and background automation.

Each workstream gets its own git worktree — your main branch stays clean, multiple workstreams run concurrently. Each task auto-commits. When a workstream is done, click Create PR.

Three steps No onboarding call

1
Clone it
Needs Node, Docker, and a Claude subscription. That's the whole dependency list.
git clone git@github.com:ilyador/codesync.git && cd codesync && pnpm install
2
Start it
Starts Supabase and the dev server. One command, two services.
npx supabase start && npx supabase db reset && pnpm dev
3
Ship something
Point it at your repo. Create a task. Click Run. Go make coffee. Come back to a PR.
open http://localhost:3000
